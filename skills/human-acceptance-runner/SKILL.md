# Human Acceptance Runner

Use when product is ready for manual walkthrough validation.

## Checklist format
- Feature name
- Preconditions
- Exact steps
- Expected result
- Failure indicators
- Pass/fail mark

## Scope
Include all user-visible features implemented so far, including realtime voice, routing, transcript, upload, vote, and deployment checks.
